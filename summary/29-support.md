# Support

## Getting help

If you run into a problem, the fastest path to a useful answer is to
gather a few details first:

- The **market and sequence number** of the order in question (this
  lets anyone trace exactly what happened — see
  [API & SDK](27-api-and-sdk.md)).
- The **transaction signature**, if you have one, so the on-chain
  result can be looked up on a block explorer.
- Your **Fermi account address** and a description of what you
  expected versus what you observed.

## Self-service first

Many questions are answered in these docs:

- Execution and fills — [Matching Engine](16-matching-engine.md),
  [Order Types](17-order-types.md).
- Margin, health, and liquidation —
  [Margin & Account Health](20-margin-and-health.md),

- Recovery when the fast path is unavailable —

- Common questions — [FAQ](28-faq.md).

## Channels

Official support and community channels, status pages, and security
contact details are published in the app. Use the official links from
the app rather than third-party sources.

## Reporting a security issue

If you believe you've found a vulnerability, report it through the
official security contact published in the app. Please do not disclose
suspected vulnerabilities publicly before they've been addressed.

## Staying updated

Network addresses, new market listings, parameter changes, and release
notes are published with each release. Pull current program IDs,
market and queue addresses, and endpoints from the published reference
rather than hard-coding them.
